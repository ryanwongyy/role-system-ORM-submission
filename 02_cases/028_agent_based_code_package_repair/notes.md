# Case 28: Controlled agent-based repair of broken social-science code packages

## Summary

Inject realistic failures into reproducible social-science code packages and compare prompt-based repair with autonomous agent repair.

## Why it matters for typology

A clean test of whether some technical reproducibility labor should be demoted to a shrinking task-cluster.

## Caveats

- Falsification test: Agent-based repair fails broadly or still requires dense manual debugging
- Main confounds: Controlled failure injection is cleaner than real-world package decay
- Evidence status: Preprint

## Sources collected

- `C028_P1` (preprint / open): Automating Computational Reproducibility in Social Science: Comparing Prompt-Based and Agent-Based Approaches — https://arxiv.org/abs/2602.08561
- `C028_S1` (accepted conference paper / open): REPRO-Bench: Can Agentic AI Systems Assess the Reproducibility of Social Science Research? — https://aclanthology.org/2025.findings-acl.1210/
